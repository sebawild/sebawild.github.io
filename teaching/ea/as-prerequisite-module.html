CS 566 as Prerequisite Module

The goal of the prerequisite version of CS 566 is to prepare you for your MSc program by closing a skill gap in your prior knowledge.

Learning outcomes

Students

  1. are able to apply common algorithm design patterns to new algorithmic problems,
  2. are able to use efficient data structures in algorithm design,
  3. are able to analyze algorithms with respect to correctness and effort,
  4. can apply scientific working methods (recognizing, formulating, solving problems, abstraction),
  5. can competently present scientific content, both in front of an audience and in a discussion.

Self-study material

A list of self-study material will be provided here. This material defines the scope of the examination for the prerequisite version of CS 566.

PRELIMINARY LIST
The material below is provided for students who want to get started early.
Material may be removed, replaced, or added until the start of the module.

The following chapters in Cormen, Leiserson, Rivest, and Stein: Introduction to Algorithms (3rd edition) are relevant for the prerequisite version of CS 566:

  • I Foundations
    • 3 Growth of Functions
    • 4 Divide-and-Conquer – Mastertheorem
  • II Sorting and Order Statistics
    • 6 Heapsort
  • III Data Structures
    • 10 Elementary Data Structures (excluding 10.2, 10.3, and 10.4)
    • 12 Binary Search Trees
    • 13 Red-Black Trees
    • 18 B-trees
  • IV Advanced Design and Analysis techniques
    • 15 Dynamic Programming
    • 16 Greedy Algorithms (excluding 16.4, 16.5)
    • 17 Amortized Analysis
  • V Advanced Data Structures
    • 19 Fibonacci Heaps
    • 21 Data Structures for Disjoint Sets (excluding 21.4)
  • VI Graph Algorithms
    • 22 Elementary Graph Algorithms
    • 23 Minimum Spanning Trees (23)
    • 24 Shortest-Source Shortest Paths (excluding 24.2 and 24.5)
    • 26 Maximum Flow (excluding 26.3, 26.4, and 26.5)
  • VII String Matching
    • 27 The naive string-matching algorithm (32.1)
    • 28 The Rabin-Karp algorithm (32.2)
    • 29 String matching with finite automata (32.3)
    • 30 The Knuth-Morris-Pratt algorithm (32.3)
